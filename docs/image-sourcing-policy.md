# 한국플레이리스트 이미지 소싱 정책

## 우선순위

1. 한국관광공사/포토코리아 이미지를 최우선으로 사용한다.
2. 한국관광공사 이미지가 부족하면 이미지 2.0으로 글의 지역, 장소, 계절, 동선에 맞는 실사형 이미지를 생성한다.
3. 마이리얼트립 이미지는 숙소 추천, 숙소 권역 비교, 호텔/숙박 상품 소개처럼 해당하는 숙소 사진이 필요한 경우에만 사용한다.

## 표시 규칙

- 한국관광공사 이미지는 `ⓒ한국관광공사 포토코리아-촬영자` 형식으로 표시한다.
- 숙소 사진 목적으로 사용하는 마이리얼트립 이미지는 `ⓒmyrealtrip`로 표시한다.
- 이미지 2.0 생성 이미지는 `ⓒ한국플레이리스트 이미지 2.0`로 표시한다.

## 이미지 2.0 생성 기준

- 반드시 실제 여행 사진처럼 보이는 실사형 이미지로 만든다.
- 일러스트, 3D, 만화, 아이콘, 과한 보정, 스톡 이미지처럼 부자연스러운 표현은 피한다.
- 프롬프트에는 지역명, 장소 유형, 계절, 시간대, 여행자 검색 의도, 필요한 구도를 넣는다.
- 실제 특정 업체, 인물, 상표, 표지판 문구, 입장권 등을 허위로 재현하지 않는다.
- 생성 이미지는 관광지 분위기, 이동 동선, 숙소 권역, 계절감처럼 글 이해를 돕는 용도로만 사용한다.

## 파일 위치

- 생성 이미지는 `public/images/generated/`에 영어 slug 파일명으로 저장한다.
- 이미지 변형은 `scripts/generate-image-variants.py`로 생성하며 `public/images/optimized/generated/`에 WebP로 저장한다.
- Markdown에서는 원본 이미지 경로를 `heroImage` 또는 본문 figure에 사용하고, 가능한 경우 `getWebpSrcSet` 패턴에 맞는 변형도 함께 제공한다.

## 품질 확인

- 한국어 글 안에서는 같은 사진을 중복 사용하지 않는다.
- heroImage와 본문 이미지는 가능한 서로 다른 이미지를 사용한다.
- 모든 이미지에는 검색 의도와 장소가 드러나는 구체적인 alt 텍스트를 넣는다.
- 빌드 전 누락 이미지, 깨진 경로, 과도하게 큰 파일, 캡션 누락을 확인한다.
